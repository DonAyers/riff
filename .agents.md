# Riff — Agent Directives

## Project Overview

Riff is a browser-based music transcription app. A musician records audio (riff, chord, arpeggio) and the app returns the detected notes and chord name. Everything runs client-side — no backend.

## Goals

1. **Core feature**: Record audio → detect all notes played (polyphonic) → display note names + chord identification
2. **No custom model training**: Use Spotify's Basic Pitch, a pre-trained ~8 MB CNN shipped via npm
3. **Cross-platform via PWA**: Runs on any device with a modern browser; installable, works offline
4. **Stay in the React ecosystem**: The developer is a React dev — minimize friction, maximize familiar tooling
5. **Performance over purity**: Favor pragmatic choices (e.g. ScriptProcessor for scaffolding) that work now, refactor later

## Architecture Decisions

- **Pitch detection**: `@spotify/basic-pitch` — polyphonic, browser-ready via TensorFlow.js, Apache 2.0
- **Music theory**: `tonal` — maps MIDI to note names, detects chords from pitch class sets
- **Audio capture**: Web Audio API with `AudioContext({ sampleRate: 22050 })` — Basic Pitch expects 22050 Hz mono
- **Model loading**: The TF.js model ships inside the `@spotify/basic-pitch` npm package under `model/model.json`
- **No backend**: All inference and audio processing happens in the browser
- **Dark theme**: Default color scheme is dark (`#0f0f0f` background)

## Lessons Learned

### Basic Pitch API
- `BasicPitch` constructor requires a model path string or `Promise<tf.GraphModel>` — it cannot be instantiated with zero arguments
- `evaluateModel()` takes `(audioBuffer, onCompleteCallback, percentCallback)` — the callback receives `(frames[][], onsets[][], contours[][])`
- `outputToNotesPoly(frames, onsets, onsetThresh, frameThresh, minNoteLen)` — note that `contours` is NOT passed to this function
- `noteFramesToTime()` takes `NoteEvent[]` (frame-based) and returns `NoteEventTime[]` (time-based with `startTimeSeconds`, `durationSeconds`, `pitchMidi`, `amplitude`)
- The model files are at `node_modules/@spotify/basic-pitch/model/model.json` and `group1-shard1of1.bin`

### Web Audio API
- `AudioContext` on iOS Safari must be created inside a user gesture handler (click/tap)
- Set `sampleRate: 22050` directly on the AudioContext to avoid manual resampling
- Disable `echoCancellation`, `noiseSuppression`, and `autoGainControl` in `getUserMedia` constraints for cleaner instrument audio
- `ScriptProcessor` is deprecated but universally supported; migrate to `AudioWorklet` when adding real-time features

### Vite + TensorFlow.js
- TF.js bundles are large (~1.2 MB minified); consider dynamic `import()` to code-split and lazy-load the model
- Vite resolves the model JSON via `new URL("@spotify/basic-pitch/model/model.json", import.meta.url).href`
- The model binary shard is automatically picked up by Vite's asset pipeline

### Tonal.js
- `Midi.midiToNoteName(60)` → `"C4"`
- `Chord.detect(["C", "E", "G"])` → `["CM", "C"]` — returns an array of possible symbols, take the first
- `Chord.get(symbol)` returns `{ tonic, quality, aliases, empty }` for formatting

## Coding Conventions

- **TypeScript strict mode** — `strict: true`, `noUnusedLocals`, `noUnusedParameters`
- **Hooks pattern**: Business logic lives in custom hooks (`useAudioRecorder`, `usePitchDetection`), components are presentational
- **lib/ folder**: Pure utility functions with no React dependencies (noteMapper, chordDetector)
- **CSS**: Component-scoped CSS files co-located with their `.tsx` files (e.g. `Recorder.css` next to `Recorder.tsx`)
- **No external CSS framework** in MVP — plain CSS with CSS variables
